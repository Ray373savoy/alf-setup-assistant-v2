## Plan Task プロセス Entry

### 目的

- **既存の相談対話履歴（message-output形式）に基づいて、ChannelTalk Taskのノード設計ダイアグラムを段階的に作成する。**
- **1段階では「現在運用中のフロー」をできる限りそのまま移した初期ダイアグラムを合意・確定する。**
- **2段階では確定したダイアグラムを基に「追加したいロジック/例外/自動化」を質問しながら高度化する。**

### 進行フロー

1. **1段階：既存フローベースラインダイアグラム生成＆確定**

   - [`node.plan.turn1.md`](./node.plan.turn1.md)
   - 入力：
     - `conversationRecords`：`poc/message-output.js`が生成する対話要約配列
     - `taskName`, `mainGoal`：どのTaskを作りたいかの一行要約
     - （任意）`keyStages`：人が見やすい大きな段階区分（例：「問い合わせ開始」「申請情報入力」「書類アップロード」「検討/完了案内」）
   - 出力：
     - **FigJam JSON（現在のフローベースライン、ALFノードタイプ注釈付き）1つ**
     - **短い確認質問**：「今のダイアグラムが実際の相談フローをうまく反映しているか」「修正すべき部分があるか」のみ聞く。
   - 目的：
     - **対話履歴をそのまま一行ずつ移すのではなく、意味のある段階/分岐に抽象化された「現状基準」ダイアグラムを確保する。**
     - 以降の2段階で高度化する「ベースダイアグラム」を確定することが目標。

2. **2段階：追加ロジック収集＆ダイアグラム高度化**
   - [`node.plan.turn2.md`](./node.plan.turn2.md)
   - 入力：
     - `baseDiagram`：1段階で最終合意した FigJam JSON
     - `taskName`, `mainGoal`（同一）
     - （任意）`conversationRecords`：追加で参考にしたい元の対話要約配列
   - 出力：
     - **高度化された最新 FigJam JSON 1つ**
     - **短い質問**：
       - 「どのノード/区間にどんな追加ロジックを付けたいか」
       - 「どの例外ケースを別途分岐したいか」
       - 「どの知識（文書/API/FAQ）を参考に自動化できるか」
   - 目的：
     - **ベースダイアグラムの上に「追加分岐、自動化コード/関数、相談員接続条件、知識ソース基盤ロジック」等を段階的に設計する。**
     - **各高度化段階ごとにダイアグラム全体を再描画し、常に最新バージョンを一目で確認できるようにする。**

### 運用指針

- **Taskスペック遵守必須**：設計時に`task/spec.md`と`task/policy.md`を必ず参照すること。
  - 特に **「ボタン（Button）ノード」は使用できない**という点を留意する。既存の対話ログにボタンがあっても、代替方法（例：選択式テキスト選択、短答型質問等）をユーザーに提案または確認すること。
- **1段階では「現在のフローを正確に移すこと」にのみ集中し、新しいロジックのアイデアはメモだけして2段階で本格的に反映する。**
- **2段階では「どこにどんなロジックを追加したいか」を具体的に聞きながらやり取りする。**
  - 例：「`C`ノード（書類案内）の後に未提出顧客をリマインドする分岐を追加しましょうか？」
  - 例：「RID申請ではなく他のメニューを間違えて選んだ顧客のための『移動/再選択』分岐を追加しましょうか？」
- ユーザーが「**確定**」「**完了**」「**done**」と言ったら、**該当段階の最後のレスポンスではダイアグラムのみ出力**し、説明/質問はしない。

### 入力テンプレート（要約）

- **taskName**：タスク名（例：「電話発信番号変更申請 Task設計」）
- **mainGoal**：このTaskが最終的に解決すべき目標（例：「RID申請をセルフ書類提出まで自動化し、検討結果案内までの状態を管理する。」）
- **keyStages（任意）**：人が理解しやすい大きな段階の列挙
- **conversationRecords（任意）**：`message-output.js`形式の対話要約配列
- **baseDiagram（2段階専用）**：1段階で合意された FigJam JSON
