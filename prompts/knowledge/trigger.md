# Trigger（トリガー）

Taskの開始条件を定義するゲートです。

## 役割

自然言語指示文と顧客フィルターに基づき、ALFが特定のTaskを開始すべきタイミングを決定します。

## テンプレート

```
[役割]
あなたは{顧客企業}の{業務}を案内するチャットボットです。

[トリガー条件]
顧客が{状況}について問い合わせたら開始します。

問い合わせ例：
- "{表現1}"
- "{表現2}"
- "{表現3}"
- "{表現4}"
...
```

**具体例（参考用）**

```
[役割]
あなたはオンラインショッピングモールの注文/配送状況を案内するチャットボットです。

[トリガー条件]
顧客が注文状態や配送状況について問い合わせたら開始します。

問い合わせ例
- "注文どこまで来ましたか？"
- "配送を確認してください"
- "いつ届きますか？"
- "荷物はどこにありますか？"
- "注文状況を確認したいです"
```

## トリガーが重要な理由

- トリガーは**自然言語で動作**します。
- 開始ルールを**具体的に設定**するほどトリガーされる確率が高くなります。
- Taskがうまく動作するためには**トリガー設定が最も重要**です。

## トリガー条件作成のコツ

FigJam/提案書/相談内容から顧客が実際に使う表現を抽出：

1. **同じ意図の多様な表現**
   - "{動作}はどうすればいいですか？" / "{動作}の方法" / "{動作}の設定"

2. **口語体、タイプミス、略語**
   - "どうすればいいですか？" / "どうするの？" / "教えてください"

3. **問題状況の表現**
   - "できません" / "うまくいかない" / "エラーが出ます"
